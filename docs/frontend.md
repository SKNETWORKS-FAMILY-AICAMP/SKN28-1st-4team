# 프런트엔드

`fe/` 는 사용자가 실제로 만나는 입력 화면과 결과 화면을 담당합니다.  
이 프로젝트에서 프런트엔드의 역할은 단순한 폼이 아니라, 차량 선택과 결과 해석을 제품 경험으로 연결하는 것입니다.

## 담당 역할

- 브랜드, 모델, 세부 트림 선택 경험 제공
- 기본 차량 정보 입력
- 현재 예측가와 향후 감가 흐름 시각화
- 상승 요인과 하락 요인을 사용자 친화적으로 표현

## 현재 페이지 흐름

1. 랜딩 페이지
2. 차량 입력 페이지
3. 결과 페이지

즉 프런트는 사용자가 "무슨 차를 가지고 있고, 지금 어떤 상태인지"를 입력하게 만들고, 그 입력을 예측 결과 화면으로 자연스럽게 연결하는 층입니다.

## 데이터 경계

프런트는 모든 데이터를 백엔드에서 받지 않습니다.

- 차량 선택용 기준 데이터
  - 로컬 JSON 자산 사용
- 모델 이미지, 가격 예측 결과, 요인 분석
  - 백엔드 API 호출

이 경계는 배포 시점에도 유지되는 운영 기준입니다.
